# ai-thread-vault

A local, searchable vault for your AI chat history. Import conversation exports
from ChatGPT, Claude, Gemini, and generic JSON/Markdown, store them in SQLite with
**FTS5 full-text search**, and query them offline.

> Generated by [Protozoa](https://smartinfer.ai) (SmartInfer Inc.).

---

## What it does

- **Import** chat exports — `chatgpt` (conversations.json), `claude`, `gemini`, `jsonl`, `markdown`.
- **Store** threads + messages in SQLite; messages are indexed in an FTS5 virtual table.
- **Search** across all your conversations with ranked, snippet-highlighted results.
- **Show / export / stats** — read a thread, export it to Markdown, count threads by platform.

Everything runs locally. No network, no accounts — it reads the *export files* those
tools produce, it never calls any AI service.

## Quick start

```bash
# Create a vault
python -m ai_thread_vault.cli init vault.db

# Import a conversation export (jsonl | chatgpt | claude | markdown)
python -m ai_thread_vault.cli import vault.db jsonl my_export.jsonl

# Search
python -m ai_thread_vault.cli search vault.db "fts5"

# Inspect
python -m ai_thread_vault.cli show   vault.db <thread_id>
python -m ai_thread_vault.cli export vault.db <thread_id>
python -m ai_thread_vault.cli stats  vault.db
```

Library use:

```python
from ai_thread_vault import schema, store, query
from ai_thread_vault.importers import jsonl
```

## Verify it works

```bash
python -m pytest        # acceptance tests: import → search round-trips per format
```

The tests drive the real CLI with tiny fixtures under a blocked network, so they
exercise the actual program end to end and never contact an AI service.
